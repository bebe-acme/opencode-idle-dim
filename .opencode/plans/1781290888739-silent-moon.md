# Plan: Multiple screensavers (smaller alien + Pac-Man + 8-bit loading bar)

## Goal

Turn the single DVD-bounce screensaver into a **pluggable screensaver system** with 3 modes, one picked **at random on each `/idle`**:

1. **Alien DVD bounce** — same as today but a **smaller** alien.
2. **Pac-Man** — chomps across the screen eating dots and **colored ghosts**.
3. **8-bit loading bar** — retro 90s PC "LOADING..." screen (white block letters + black-and-white bordered bar with chunky green segments, matching the reference image), loops.

Plus a **persistent project-identity header** (name + folder) shown on top of every saver, so a parked session is always identifiable.

All keep the current safety guarantees: rendered in the `app` slot overlay (no `api.route`), only paints while dimmed, dismissable by any key / ⌘K / `/active`.

## Architecture: pluggable savers

Replace the single bounce with a list of saver objects. Each saver owns its state and three methods:

```js
// factory → instance with closure state
function makeAlienSaver() {
  let x, y, dx, dy, color
  return {
    name: "alien",
    stepMs: 110,
    reset(w, h) { /* init position/velocity */ },
    tick(w, h)  { /* advance + bounce + color flip */ },
    render(w, h) { return [ /* absolute-positioned el() nodes */ ] },
  }
}
function makePacmanSaver()   { /* ... */ }
function makeProgressSaver() { /* ... */ }
```

In `tui()`:

```js
const savers = [makeAlienSaver(), makePacmanSaver(), makeProgressSaver()]
let activeSaver = null
const pickSaver = () => {
  activeSaver = savers[Math.floor(Math.random() * savers.length)]
  const { w, h } = termSize()
  activeSaver.reset(w, h)
  log(`saver: picked ${activeSaver.name}`)
}
```

- `startSaver()` reads `activeSaver.stepMs` for the interval; each tick calls `activeSaver.tick(w,h)` then `bumpAnim`.
- App-slot `children` getter: `if (!isDim() || !activeSaver) return []; animTick(); return [...activeSaver.render(w,h), hint]` (the "press any key / ⌘K" hint stays common to all savers).
- `apply()` idle-entry branch: `pickSaver(); startSaver()`. Re-idle guard: `if (isDim() && !saverTimer) { if (!activeSaver) pickSaver(); startSaver() }`. Wake/heal: `stopSaver(); activeSaver = null`.

This keeps one interval, one signal, one overlay; only the active saver's logic differs.

## Common: project identity header (CRITICAL — always visible)

The overlay covers the sidebar, hiding the orange `sidebar_title`. So the user can no longer tell **which parked session/folder** they're looking at. Fix: render a **persistent identity header at the top of the overlay**, shown for **all three savers**, in bright orange.

Two lines, top-center (or top-left), rendered **last** so it sits on top of the animation:

```
▶ DIMM
  ~/Desktop/playground_beib/opencode-idle-dim
```

- **Project name**: capture `props.title` from the `sidebar_title` slot into a closure var `lastTitle` each time that slot renders (the host passes the real title there). Use it in the overlay header.
- **Folder**: `api.state?.path?.directory || api.state?.path?.worktree || process.cwd()`, abbreviated with `~` for `homedir()`. If wider than the terminal, truncate the middle (`…`) keeping the tail (basename matters most).
- Defensive: wrap `api.state` access in try/catch; fall back to `lastTitle` only, or `process.cwd()`.
- Reserve the **top 3 rows** for the header so savers don't overlap it: saver animation area is `y ∈ [3, h)`.

This header is added in the common `app`-slot wrapper alongside the bottom hint, independent of which saver is active.

## Saver 2 — Pac-Man with ghosts

Horizontal arcade scene centered vertically (`row = floor(h/2)`).

- **Pac-Man**: full-block sprite (proven to render), yellow `#ffd400`, 2-frame chomp (mouth open ↔ closed) alternating each tick. ~6 cols × 5 rows. Moves right at ~1 col/tick.
- **Track**: a line of dots `·` (dim) spaced every 2 cols across the row, plus **ghosts** at a few positions. Ghosts are small full-block sprites (~4×3) each a classic color: Blinky `#ff0000`, Pinky `#ffb8ff`, Inky `#00ffff`, Clyde `#ffb851`.
- **Eating**: anything at/behind Pac-Man's mouth column is removed (dots blank, ghosts vanish). Optional: brief "eaten" flash.
- **Loop**: when Pac-Man passes the right edge (or eats everything), `reset()` re-lays dots+ghosts and starts again from the left, optional new row.
- `stepMs ≈ 140`.

State: `pacX`, `mouthOpen`, `dots[]` (cols with a dot), `ghosts[]` (`{col,color,alive}`).
`render`: dots as `·` text nodes, alive ghosts as colored sprites, Pac-Man sprite at `pacX`, all absolute-positioned on the center row. Sprites finalized in code (full-block style like the alien; draft below).

## Saver 3 — 8-bit loading bar (90s PC, match reference image)

Recreate the reference: chunky pixel **"LOADING..."** in **white**, a **black-and-white bordered bar** filled with **chunky green segments** (vertical bars with gaps), B&W + green palette. State: `pct` 0→100, `dots` (0–3 for the `...` animation), `phase` ("loading" | "done"), `blink`.

Layout, centered (`left = floor((w - barTotalW)/2)`, `top = floor(h/2) - 3`):

1. **"LOADING..." in block letters** — hand-crafted 5-row pixel font using `█` in **white** (`#ffffff`), with the trailing `.` `..` `...` cycling via `dots`. (Reliable; do not depend on opentui `ascii_font`.) Draft for the word height ~5 rows; letters from a small block-font table `L O A D I N G`.
2. **Segmented bar** — a bordered rectangle of ~12 segment cells. Border drawn in **white** with box-drawing chars (`┌─┐│└┘` or heavy `┏━┓┃┗┛`). Inside, each segment is a chunky green block (`█` or `▮`, 1 col wide) separated by a 1-col gap, filling left→right as `pct` rises:

   ```
   ┏━━━━━━━━━━━━━━━━━━━━━━━━━┓
   ┃ █ █ █ █ █ █ █ ░ ░ ░ ░ ░ ┃
   ┗━━━━━━━━━━━━━━━━━━━━━━━━━┛
   ```
   Filled segments green `#3bdc3b`/`#00ff2a`, empty segments dim (`#1a1a1a`). `segments = 12`; `filled = round(pct/100*segments)`.

- `tick` (stepMs ≈ 140): `pct += ~3`; advance `dots` every few ticks. At `pct >= 100`: `phase="done"` for ~1.5s showing a blinking `▸ PRESS ANY KEY` under the bar, then `reset()` (pct 0, new state) → loops.
- `render`: the white block-letter "LOADING" rows, the bordered segmented bar rows, optional blinking done-line; all absolute-positioned. Palette: white + green only (matches the 90s B&W aesthetic), no extra colors.

Note: the identity header (project name + folder, orange) still renders on top regardless — so even the loading screen shows which session it is.
