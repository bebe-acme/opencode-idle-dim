# Plan — Two iTerm2 tab commands: smart per-tab color + auto-name

## Goal (from Beib)
Beib runs **two opencode sessions per iTerm tab** (split panes). Two new on-demand
opencode slash-commands:

1. **Smart tab color** — a tab must NOT look parked when only one pane is idle.
   Decision: a tab is **teal (`#529e99`) if ANY pane is active**, **grey (`#2b2b2b`)
   only if ALL panes are idle**. (Beib explicitly chose "color inteligente por tab",
   NOT physically migrating panes — AppleScript can't move panes and the iTerm2
   Python API route is heavy/disruptive.)
2. **Auto-name** — set each tab's name to `proj1:proj2` (the two panes' project
   labels joined with `:`). Decision: label = **basename of the project folder**.
   Decision: **on-demand slash-commands** (no background watcher).

## Key feasibility findings (researched)
- iTerm2 **AppleScript cannot move a session/pane between tabs** (maintenance mode).
  Only the iTerm2 **Python API** can (`Window.async_set_tabs`, "Move Session to Tab"
  menu id) — needs enabling the API + `pip install iterm2 pyobjc`, runs async, and
  reshuffles layout. **Rejected** per Beib's choice; not needed for the chosen design.
- Tab background color is set by **OSC 6** (`\033]6;1;bg;{red,green,blue};brightness;N\a`)
  written to a session's TTY — already proven working in this repo (`paintTab` /
  manual painting). With 2 panes writing different colors, the tab color is ambiguous
  → that's the root bug the color command fixes by painting **all panes of a tab the
  same computed color**.
- Session/tab **name** is set by **OSC 1** (`\033]1;NAME\a`) written to the TTY
  (iTerm "Session Name"). Writing it to both panes makes the tab read `p1:p2`
  regardless of focus — exactly what Beib does by hand today.
- `bin/opencode-iterm-state list` already emits `window=W tab=T session=S name=… tty=…`
  for every session → enough to group panes by tab in bash. No new AppleScript needed.

## Architecture decision: ZERO plugin changes
Both commands are **pure additions** to `bin/opencode-iterm-state` + two command files.
They read the existing flag files (`~/.local/state/opencode-idle/<tty>.flag` = idle)
and the existing `list` AppleScript, then write OSC codes to each TTY — the same
mechanism already used. The plugin's existing per-session auto tab-paint (commit
`e823465`) **stays** (Beib likes the auto teal/grey for single-pane tabs). For split
tabs, `/tabs-color` is the tab-aware on-demand fix; its writes override the plugin's.
(Trade-off: after changing a pane's state in a split tab, re-run `/tabs-color`. This
matches the chosen on-demand model. A future Phase 2 could auto-run it via a watcher.)

## Files to create / modify
- `bin/opencode-iterm-state` — add two subcommands `tabs-color` and `tabs-name`
  (+ update `usage()` and the `case` dispatcher). Reuse `list_sessions`.
- `command/tabs-color.md` — `/tabs-color` → runs `~/.local/bin/opencode-iterm-state tabs-color`
- `command/tabs-name.md` — `/tabs-name` → runs `~/.local/bin/opencode-iterm-state tabs-name`
- `install.sh` — `install` the two new `command/*.md` files.
- `README.md` + `handoff.md` — document both commands, the teal/grey rule, basename
  naming, the split-tab re-run note, and the rejected migration approach.

## Implementation details

### `bin/opencode-iterm-state tabs-color`
- Constants: `TEAL_RGB="82 158 153"`, `GREY_RGB="43 43 43"` (match plugin
  `TAB_ACTIVE`/`TAB_IDLE`). `FLAG_DIR=~/.local/state/opencode-idle`.
- Capture `list_sessions` output. Parse each line via parameter expansion
  (`tty=${line##*tty=}`, `name=${line#*name=}; name=${name% tty=*}`,
  `key=${tty##*/}`, `group="win:tab"` parsed similarly).
- Opencode-pane detection: name contains `(node)` **OR** a `<key>.flag` exists.
  (Skip `btop`, `ssh`, etc.)
- Use bash associative arrays: `group_ttys[group]` (space-joined ttys),
  `group_active[group]=1` if any opencode pane in the group has **no** flag.
- For each group with opencode panes: `color = TEAL if group_active else GREY`;
  write OSC 6 to **every** opencode tty in that group:
  `printf '\033]6;1;bg;red;brightness;%d\a\033]6;1;bg;green;brightness;%d\a\033]6;1;bg;blue;brightness;%d\a' R G B > "$tty"`.
- Print a summary line per painted tab (e.g. `TEAL win2 tab9  ttys003 ttys008`).

### `bin/opencode-iterm-state tabs-name`
- Parse + group by tab the same way; keep panes in `session=` order.
- Per opencode pane, resolve the project dir → basename:
  1. Find the opencode pid on the tty: `ps -t "$key" -o pid=,command= | awk '/opencode/{print $1; exit}'`.
  2. cwd via `lsof -a -p "$pid" -d cwd -Fn 2>/dev/null | sed -n 's/^n//p' | head -1`.
  3. `label=$(basename "$dir")`; skip pane if unresolved.
- `combined`= labels joined with `:` (single pane → just the one label).
- Write OSC 1 to **every** opencode tty in the group:
  `printf '\033]1;%s\a' "$combined" > "$tty"`.
- Print a summary line per tab (e.g. `NAME ttys003 ttys008 -> opencode-idle-dim:FINANCES`).

### Command files (mirror `command/idle.md` style)
Frontmatter `description:` + a body instructing Bash to run the binary and report
output concisely.

### `install.sh`
Add `install -m 0644 "${REPO_DIR}/command/tabs-color.md" "${OC_DIR}/command/tabs-color.md"`
and the same for `tabs-name.md`.

## Risks & mitigations
- **opencode may re-set the session title.** Some tabs show `OC | …` (opencode-set);
  Beib's manual renames persisted, suggesting it doesn't continuously override.
  Verify during impl: set a name, change focus / trigger a render, confirm it sticks.
  If it gets clobbered, mitigation = re-run `/tabs-name`, or check for an opencode
  config to disable terminal-title integration.
- **`lsof` cwd detection** can mis-pick the pid on a busy tty. Mitigation: match
  `opencode` in `command=`; if multiple, take the first; if unresolved, skip that
  pane (don't write a wrong name). (Optional later: have the plugin publish
  `<tty>.info` with `api.state.path.directory` for an exact source.)
- **Plugin vs command interaction** in split tabs: the plugin paints per-session on
  state change; `/tabs-color` overrides on demand. Documented; re-run after changing
  a pane's state. (Phase 2: plugin could shell out to `tabs-color` on state change.)
- **Writing OSC to other TTYs** needs same-user permission — already confirmed (the
  manual painting earlier wrote to every session's `/dev/ttysNNN`).
- **`(node)` heuristic** could miss a pane momentarily running a non-node job; the
  `flag`-exists OR-clause covers idle panes regardless.

## Verification (end-to-end)
1. `node --check bin/...` n/a (bash) → `bash -n bin/opencode-iterm-state`.
2. Deploy: `cp` bin to `~/.local/bin/opencode-iterm-state`, `install.sh` (or cp command files). No opencode restart needed (no plugin change).
3. `~/.local/bin/opencode-iterm-state tabs-color`:
   - Split tab with one pane idle + one active → **tab stays teal**.
   - Split tab with both panes idle → **tab grey**.
   - Single-pane active → teal; single-pane idle → grey. `btop`/`ssh` untouched.
4. `~/.local/bin/opencode-iterm-state tabs-name`:
   - Split tab (e.g. opencode-idle-dim + BOMBO/FINANCES) → both panes named
     `opencode-idle-dim:FINANCES`; tab shows it.
   - Single pane → just its basename.
   - Re-run after focus change to confirm names persist (title-overwrite risk).
5. Plugin test still green (no plugin change, but run anyway):
   `node --import ./test/register.mjs --test test/idle-dim.test.mjs`.
6. Commit + push to `main`; update README + handoff.
