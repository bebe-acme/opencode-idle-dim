# Spec: Idle Fun Mode — v2

Date: 2026-06-12. Status: design approved. Builds on v1 (verified, working).

## Problem solved

v1 dimmea la TUI correctamente pero es puramente funcional. El usuario (Beib) quiere:
- Cosas divertidas en modo idle: ASCII art, humor, personalidad.
- Una transición idle→active más visible y placentera (despertar gradual).
- El logo ACME (alien verde) convertido a ASCII "volando" por la terminal.

## Dream state

Cuando una sesión de OpenCode entra en idle, la terminal muestra el marciano ACME verde flotando por la pantalla, alternando con frases random y emojis. Al volver con `/active`, la TUI despierta con un fade suave de ~1.5s que pasa por 4 niveles de brillo. Se siente como "despertar".

## Who benefits

- Beib (usuario directo): sesiones idle con personalidad, transición placentera.
- Cualquier usuario que instale el plugin: idle mode con carácter.

## Architecture

### Files

```
~/.config/opencode/
├── plugin/idle-dim.js          ← crece (~250 líneas)
├── themes/
│   ├── beib-dim.json           ← existe (factor 0.15)
│   ├── beib-dim-03.json        ← NUEVO (factor 0.30)
│   ├── beib-dim-05.json        ← NUEVO (factor 0.50)
│   └── beib-dim-07.json        ← NUEVO (factor 0.70)
├── command/idle.md              ← sin cambios
├── command/active.md            ← sin cambios
└── tui.json                     ← agrega options al plugin
```

### Plugin changes (`idle-dim.js`)

**Current behavior (keep all):**
- Watch flag file via `fs.watch` + 1.5s poll
- Theme switch to `beib-dim` on idle
- `sidebar_title` slot with orange `#ff9a00`
- Self-healing on restart without flag

**New behavior:**

1. **Content pool** — array inline de ~12 entradas:
   - ACME alien ASCII art (verde `#00ff2a`)
   - Frases random (`"beib.exe has stopped responding"`, `"afk but the vibes remain"`, etc.)
   - Emoji combos (`"（◎−◎；）zZz"`, `"💤 dreaming..."`)
   - Cada entrada tiene `type` (ascii/emoji/phrase) y `duration` sugerido

2. **Route-based idle screen (primary path):**
   - Register custom route `idle` via `api.route.register()`
   - Route renders: alien/frase/emoji en posición random, centrado
   - Background: dim theme already active
   - Rotation: `setInterval` ~10s cambia posición + contenido
   - On active: `api.route.navigate()` back to session, then fade sequence

3. **Slot fallback (if route fails):**
   - `sidebar_content`: mini alien + "💤 idle — /active to wake"
   - `sidebar_title`: emoji random + orange title (existing, enhanced)
   - `session_prompt_right`: subtle emoji indicator
   - Rotation via `setInterval` on sidebar slots

4. **Wake-up fade sequence:**
   - On flag removal, step through intermediate themes:
     ```
     0ms: beib-dim (0.15) → 400ms: beib-dim-03 (0.30) → 800ms: beib-dim-05 (0.50)
     → 1200ms: beib-dim-07 (0.70) → 1500ms: original theme
     ```
   - Each step: `api.theme.set(theme)` with 400ms delay
   - Total: ~1.5s gradual brightness ramp

5. **Alien "flight":**
   - `setInterval` of ~10s (randomized 8-12s)
   - On each tick: pick random position (top/left padding), render content
   - Alien appears ~60% of rotations, phrases ~30%, emoji ~10%

### Theme generation

Intermediate themes generated by multiplying the original palette by the target factor. Each theme is a static JSON file in `themes/`. Pre-generated, not dynamic.

Factors:
- `beib-dim.json`: 0.15 (existing, ultra-dark)
- `beib-dim-03.json`: 0.30
- `beib-dim-05.json`: 0.50
- `beib-dim-07.json`: 0.70

### Unchanged

- `bin/opencode-iterm-state`: flag creation/removal, iTerm2 tab tint — no changes.
- Commands `/idle` and `/active`: same behavior.
- Install script: updated to copy new theme files.

## Acceptance criteria

1. **ACME alien** renders correctly in OpenCode TUI during idle, in green.
2. **Alien "flies"**: position changes every ~10s during idle.
3. **Content rotates**: alien, phrases, and emojis alternate over time.
4. **Wake-up fade**: restoring from idle shows 4-step brightness ramp over ~1.5s.
5. **Route primary path**: if `api.route.register` works, full-screen idle overlay. If not, slot-based fallback renders correctly.
6. **Existing behavior preserved**: flag file, per-TTY targeting, sidebar title orange, idempotence, self-healing — all unchanged.
7. **No regressions**: `node --check` clean, `bash -n` clean, no SIGUSR2 usage, no ANSI palette dimming.
8. **Triple idle/active cycle**: verified visually in disposable OpenCode instance.

## Risks and mitigations

| Risk | Mitigation |
|------|-----------|
| `api.route.register` loses session state | Fallback to slot-based rendering; route is optional |
| Intermediate themes don't feel smooth | Test with 3 steps if 4 is too slow; adjust timing constants |
| Rotation interval conflicts with 1.5s poll | Single master loop managing both concerns |
| Unicode block chars (█) don't render in all terminals | Test in iTerm2 (confirmed); document fallback to `#` |
| Plugin grows beyond maintainable size | Keep content pool separate if > 30 lines; inline for now |
